**chirp/src/components/auth/LoginForm.tsx:**

This is a client component that renders a login form with email and password fields.

Structure and behavior:
- Uses `useForm<LoginInput>` from react-hook-form with `zodResolver(loginSchema)`
- Form mode is "onSubmit" (per UI-SPEC: on-submit validation, NOT on blur or on change)
- On submit, calls `signIn("credentials", { email, password, redirect: false })` from `next-auth/react`
- If `signIn` returns `{ ok: false }`, display "Invalid email or password" as a form-level error (per D-04 -- generic, does NOT reveal which field is wrong)
- If `signIn` returns `{ ok: true }`, call `router.push("/feed")` using `useRouter` from `next/navigation`
- While submitting, button shows a CSS spinner and all inputs are disabled (per UI-SPEC Loading State)
- Uses `useState` for `isSubmitting` to track loading state and `serverError` for the generic login error
- The generic login error is displayed above the submit button, not attached to a specific field

Form field details per UI-SPEC Copywriting Contract:
- Email field: label "Email", placeholder "you@example.com", type="email"
- Password field: label "Password", no placeholder, type="password"

Button: text "Log in", full width

Link below form: "Don't have an account? Sign up" -- links to "/signup" using next/link. "Sign up" portion is accent-colored.

Styling per UI-SPEC:
- Each field: label (text-sm font-semibold text-slate-900 mb-2) + input (h-11 px-4 border border-gray-300 rounded-lg bg-gray-50 text-base, focus:border-accent focus:ring-2 focus:ring-accent/20 focus:bg-white, error:border-red-500 error:ring-2 error:ring-red-500/20) + error message (text-sm font-semibold text-red-500 mt-1)

- Every input has an associated `<label htmlFor="fieldId">` and `id` on the input

- Uses `useForm<SignupInput>` from react-hook-form with `zodResolver(signupSchema)`
- Form mode is "onSubmit"
- On submit:
  1. POST to `/api/register` with `{ email, username, password }`
