**Step 2: Create Zod validation schema for post creation** at `chirp/src/lib/validations/post.ts`.

**Step 3: Create post service layer** at `chirp/src/lib/services/posts.ts`.
Import `db` from `@/lib/db/client`, `posts` and `users` from `@/lib/db/schema`, and `eq`, `desc`, `isNull`, `and`, `count` from `drizzle-orm`.

Define `const POSTS_PER_PAGE = 20;`

Implement these functions:

1. `getFeedPosts(page: number = 1)` — Returns top-level posts (where `parentId IS NULL`) with author info. Uses `db.select({ id: posts.id, content: posts.content, createdAt: posts.createdAt, userId: posts.userId, authorUsername: users.username, authorDisplayName: users.displayName }).from(posts).innerJoin(users, eq(posts.userId, users.id)).where(isNull(posts.parentId)).orderBy(desc(posts.createdAt)).limit(POSTS_PER_PAGE).offset((page - 1) * POSTS_PER_PAGE).all()`. This single JOIN query avoids N+1 (per POST-03, D-06).

2. `getUserPosts(username: string, page: number = 1)` — Same select shape as getFeedPosts but with WHERE condition: `and(eq(users.username, username), isNull(posts.parentId))`. For PROF-01.

3. `getUserPostCount(username: string)` — Returns the count of top-level posts by username. Use `db.select({ value: count() }).from(posts).innerJoin(users, eq(posts.userId, users.id)).where(and(eq(users.username, username), isNull(posts.parentId))).get()` and return `result?.value ?? 0`. For D-11 (post count on profile header).

4. `getUserByUsername(username: string)` — Returns user by username WITHOUT passwordHash. Use `db.select({ id: users.id, username: users.username, displayName: users.displayName, createdAt: users.createdAt }).from(users).where(eq(users.username, username)).get()`. CRITICAL: Do NOT select passwordHash (per Pitfall 3 in research).

5. `createPost(userId: number, content: string)` — `db.insert(posts).values({ userId, content }).returning().get()`. For POST-01.

6. `deletePost(postId: number, userId: number): boolean` — `const result = db.delete(posts).where(and(eq(posts.id, postId), eq(posts.userId, userId))).run(); return result.changes > 0;`. CRITICAL: The WHERE clause MUST include both `posts.id` AND `posts.userId` for ownership check (per POST-02, D-10, Pitfall 2 in research).

Export all 6 functions. Also export the `POSTS_PER_PAGE` constant. Also export a TypeScript type for the post shape returned by getFeedPosts:
```typescript
export type FeedPost = {
  id: number;
  content: string | null;
  createdAt: string;
  userId: number;
  authorUsername: string;
  authorDisplayName: string | null;
};
```

**Step 1: Create GET/POST route handler** at `chirp/src/app/api/posts/route.ts`.

Follow the pattern established in `chirp/src/app/api/register/route.ts` (import NextResponse, parse body, Zod validate, db operation, return JSON).

```typescript
import { NextResponse } from "next/server";
import { getSession } from "@/lib/auth/session";
import { createPostSchema } from "@/lib/validations/post";
import { createPost, getFeedPosts } from "@/lib/services/posts";
```

`GET` handler:
- Call `await getSession()`. If no session, return `NextResponse.json({ error: "Unauthorized" }, { status: 401 })`.
- Parse `page` from `new URL(request.url).searchParams.get("page")`, default to 1, convert to Number.
- Call `getFeedPosts(page)` and return `NextResponse.json(posts)`.

`POST` handler:
- Call `await getSession()`. If no session, return 401.
- Parse body with `await request.json()`, validate with `createPostSchema.safeParse(body)`.
- If validation fails, return `NextResponse.json({ error: result.error.flatten().fieldErrors }, { status: 400 })`.
- Extract userId: `const userId = Number((session.user as any).id)` (same cast pattern used throughout Phase 1).
- Call `createPost(userId, result.data.content)` and return `NextResponse.json(post, { status: 201 })`.
- Wrap in try/catch, return 500 on unexpected errors.

**Step 2: Create DELETE route handler** at `chirp/src/app/api/posts/[id]/route.ts`.

IMPORTANT — Next.js 16 breaking change: `params` is a Promise. Use `RouteContext` type helper:
```typescript
import { NextResponse } from "next/server";
import type { RouteContext } from "next/types";
import { getSession } from "@/lib/auth/session";
import { deletePost } from "@/lib/services/posts";
```

NOTE: If `RouteContext` is not available from `next/types`, check the Next.js 16 docs in `chirp/node_modules/next/dist/docs/01-app/01-getting-started/15-route-handlers.md` for the correct type. The key requirement is that `params` must be awaited: `const { id } = await ctx.params`.

`DELETE` handler signature: `export async function DELETE(_request: Request, ctx: RouteContext<"/api/posts/[id]">)`
- Call `await getSession()`. If no session, return 401.
- `const { id } = await ctx.params` (MUST await — Next.js 16 async params per Pitfall 4).
- `const userId = Number((session.user as any).id)`.
- `const deleted = deletePost(Number(id), userId)`.
- If `!deleted`, return `NextResponse.json({ error: "Post not found or not authorized" }, { status: 404 })`.
- Return `NextResponse.json({ success: true })`.
- Wrap in try/catch.

"use client";
import { QueryClient, QueryClientProvider } from "@tanstack/react-query";
import { useState } from "react";

export function QueryProvider({ children }: { children: React.ReactNode }) {
  const [queryClient] = useState(() => new QueryClient({
    defaultOptions: {
      queries: {
        staleTime: 30 * 1000,
        refetchOnWindowFocus: false,

}
